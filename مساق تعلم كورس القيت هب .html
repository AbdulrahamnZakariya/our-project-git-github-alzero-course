توثيق دورة Git و GitHub

ما الذي تعلمته؟

  • إنشاء حساب على GitHub: الخطوة الأولى لإنشاء مستودعات برمجية احترافية.
  • إنشاء مستودع (Repository): منصة لتخزين وتنظيم المشاريع البرمجية.
  • التنقل عبر cd: الوصول إلى المجلدات عبر الطرفية بسهولة.
  • إنشاء ملفات: باستخدام mkdir لإعداد مجلد جديد.
  • استنساخ مستودعات: باستخدام git clone لتنزيل نسخة محلية.
  • إعداد اتصال آمن (SSH):
    1. إنشاء مفتاح: ssh-keygen.
    2. إضافة المفتاح إلى حسابك: نسخ المفتاح بـ cat ~/.ssh/id_rsa.pub وإضافته.
    3. اختبار التهيئة: ssh -T git@github.com.
  • إنشاء برانش مؤقت وإجراء Pull Request:
    1. التأكد من البرانش النشط:
      تحقق من أنك على البرانش الرئيسي master أو main باستخدام:
      git branch. إذا لم تكن على البرانش المطلوب، استخدم:
      git checkout master أو git checkout main.
    2. إنشاء برانش جديد:
      استخدم الأمر:
      git checkout -b اسم-البرانش-الجديد.
      مثال: git checkout -b feature-update.
    3. إجراء التعديلات:
      قم بتحرير الملفات وحفظ التغييرات.
    4. إضافة الملفات المعدلة:
      قم بإضافة جميع الملفات المعدلة باستخدام:
      git add .
    5. تسجيل التعديلات (Commit):
      استخدم الأمر التالي مع رسالة توضح التعديلات:
      git commit -m "رسالة توضح التعديلات"
    6. رفع البرانش الجديد:
      ارفع البرانش إلى المستودع باستخدام الأمر:
      git push origin اسم-البرانش.
      مثال: git push origin feature-update.
    7. فتح طلب Pull Request:
      1. اذهب إلى صفحة المستودع على GitHub.
      2. ستجد إشعارًا يظهر بجانب البرانش الجديد بعنوان "Compare & pull request". اضغط عليه.
      3. قم بمراجعة التعديلات وإضافة وصف واضح لما قمت به.
      4. اضغط على "Create pull request" لإرسال الطلب.
